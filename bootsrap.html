üöÄ Part 1: Introduction to Distributed Programming

Distributed programming refers to software that runs across multiple computers (nodes), often over a network. Think about how Google Docs allows several users to edit a document at the same time‚Äîthat's distributed computing at work.

  • Nodes: Independent machines or systems working together.
  • Communication: Through protocols like HTTP or message-passing systems.
  • Scalability: Easily handles growing users or workloads.
  • Fault Tolerance: If one node fails, others can continue running.

Distributed systems are all around us‚Äîin search engines, cloud apps, and collaborative tools. Building distributed apps starts with a solid understanding of web technologies. That‚Äôs where HTML, CSS, JavaScript, and frameworks like Bootstrap come in.

üåç Part 5: Ensuring Browser Compatibility

What is browser compatibility? It means your website looks and works the same in all major browsers.

  • CSS Reset: Normalizes styles.
  • Feature Detection: Checks for HTML5/CSS3 support.
  • Progressive Enhancement: Supports older browsers.
  • Vendor Prefixes: Browser-specific styles (e.g., -webkit-).

Use tools like BrowserStack, LambdaTest, or Sauce Labs to test compatibility.

Bootstrap Helper Classes

Bootstrap's helper classes are utility classes that streamline styling without custom CSS. They cover spacing, text alignment, display, colors, and more.

Common Helper Classes
  • Text Alignment: .text-start, .text-center, .text-end
  • Display Properties: .d-none, .d-block, .d-flex
  • Spacing Utilities: .m-3, .p-2
  • Colors: .bg-primary, .text-white
  • Borders: .border, .border-top
Example
<div class="m-3 p-2 bg-primary text-white border">
  Content with margin, padding, background, and border
</div>
Best Practices
  • Understand purpose before applying
  • Avoid overusing utility classes
  • Combine selectively or use custom CSS if needed
  • Refer to official documentation

Bootstrap Icons

Bootstrap Icons is a library of scalable vector icons that integrate seamlessly with Bootstrap.

Including Icons
<link href="https://cdn.jsdelivr.net/npm/bootstrap-icons@1.10.0/font/bootstrap-icons.css" rel="stylesheet">

<i class="bi bi-envelope"></i> Email
Customization
  • Adjust size via style="font-size: 2rem;"
  • Change color with text color classes (e.g., text-success)
  • Use aria-label for accessibility

Best Practices
  • Provide alternative text for screen readers
  • Optimize loading by using only needed icons
  • Keep your icons library up-to-date

Bootstrap Cards

Cards are flexible content containers with multiple variants for headers, footers, images, and content.

Basic Card Example
<div class="card">
  <img src="image.jpg" class="card-img-top" alt="Image description">
  <div class="card-body">
    <h5 class="card-title">Card Title</h5>
    <p class="card-text">Some example text.</p>
    <a href="#" class="btn btn-primary">Go somewhere</a>
  </div>
</div>
Types of Cards
  • Image cards
  • Text cards
  • Cards with lists and links
  • Cards with headers and footers
Best Practices
  • Use meaningful alt text for images
  • Keep card content concise and readable
  • Utilize cards for consistent UI design
